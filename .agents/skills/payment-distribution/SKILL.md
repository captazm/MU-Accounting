# Payment Distribution Hub

The Payment Distribution Hub is a 5-stage workflow that processes auto-generated payroll records (from Reconciliation) into final crew payments with full audit trail.

## Status Progression

```
Pending → ReadyForApproval → Approved → Processed → Paid
```

## Tab 1 — ✏️ Edit & Submit (Accountant)

Pending records are auto-generated by Reconciliation. Accountant must edit before submitting.

### Edit Modal Sections

**Header info bar:** Bill Ref, Vessel, Period, Days on Board.

**A. Earnings (Crew Receives) — editable**
- Basic Salary (default from registry)
- Bonus, PDE Fees, Visa Fees, Working Gear, Leave Pay (Refunded)

**B. Deductions — editable**
- DEP Fees / Month, Paid DEP Fees, Balance DEP Fees, DEP Fee Deducted, Leave Pay

**C. Adjust for Actual Payment**
- Actual Gross Amount (USD)
- Bank Charge (USD)
- Other Deductions
- Extra Bonus / Allowance

**Final Net Pay Preview** — computed live:
`Net = Gross + Bonus − BankCharge − OtherDed`

**Bank Details** — Bank Name, Account No, Account Name (editable).

**Action:** `[✓ Submit for Admin Approval]` → status: `ReadyForApproval`.

## Tab 2 — Admin Approval

- Admin reviews ReadyForApproval records.
- Table shows: Crew, Vessel, Gross, Bank Charge, Deductions, Bonus, Net Total, Bank, Remark.
- Admin clicks **Approve** → status: `Approved`. Accountant cannot self-approve (enforced by Firestore rules).

## Tab 3 — Distribution

Approved payments are ready to be processed.

Per-row actions:
- **🖨️ Slip** — Generates full payslip in new browser window. Includes:
  - Crew Information grid
  - Salary & Earnings (uses registry `salary`, NOT `ownerPaid`)
  - Departure (DEP) Fees Summary: Total / Previously Paid / Deducted This Month / Remaining
  - Leave Pay Summary: Deducted + Accumulated balance
  - Net Pay Calculation (USD) with bank charge logic (5 USD, exempted if salary < $200)
  - **MMK Conversion** section using rate from Payroll Reporting state
  - Payment Details (method, bank info)
  - Signature lines (Prepared / Approved / Crew)
- **✂️ Split** — Distribute one payment across multiple registered bank accounts.
  - Loads crew's banks array.
  - Enter USD amount per account; total must match exactly.
  - Each split creates a separate crewPayment doc with `-S1`, `-S2`, etc. suffix.
- **Process Done** → status: `Processed`.

## Tab 4 — Verification (Signed Slip Upload)

Processed payments require signed slip proof before being marked Paid.

Per-row UI:
- **Slip column:**
  - Not uploaded: ⚠ Missing
  - Uploaded: 👁️ View Slip + ✓ Uploaded badge
- **Action column:**
  - **📎 Upload Slip** (when missing) — native file picker, accepts image or PDF.
    - Uploads to Firebase Storage: `signedSlips/{paymentId}_{timestamp}_{filename}`
    - Auto-marks the record with `signedSlipUrl`.
  - **🔄 Replace** (when uploaded) — re-upload to overwrite.
  - **✓ Mark Paid** (Admin only, requires uploaded slip) → status: `Paid`.
  - Non-admin sees "Waiting for admin" after upload.

### Mark Paid — Auto Crew Registry Updates

When Admin clicks Mark Paid, the system updates the crew's registry record:
```
paidDepFees       += depFeeDed
balanceDepFees    -= depFeeDed     (clamped to 0)
accumulatedLeavePay += actLeavePay
```

This keeps crew DEP balance and leave pay accumulator in sync over time.

## Tab 5 — Paid History

- Lists all Paid records for the selected salary month.
- Read-only audit log: Crew Name, Amount, Paid Date.

## Manual Payment

Top-right button: **➕ Manual Payment** — for ad-hoc distributions (e.g., emergency leave pay).
- Select crew, enter amount, optional vessel override.
- Creates `CPAY-MANUAL-{timestamp}` with slipId = "AD-HOC", status = Pending.
- Flows through normal 5-stage workflow.

## Payroll Reporting (Below tabs)

Bill-grouped cards show progress visually.

**Per-bill card:**
- Header: Client name, month, bill ID, vessel
- ✓ Complete badge (when all submitted) OR `x/n submitted` counter
- **Crew Avatar Dots** — 22px circle per bill crew:
  - 🟢 Paid (green, glow)
  - 🔵 Approved/Processed (blue, glow)
  - 🔵 Submitted (cyan, glow)
  - ⬜ Pending (dashed outline, dim)
- Hover tooltip: name + status
- **Submitted crew chips** — colored with status label
- **Pending crew chips** — dashed border, "Pending" label
- **[View Report]** opens per-crew breakdown modal

**Per-crew breakdown modal:**
- 3-column card per crew (Earnings / Deductions / Net Pay & MMK)
- Editable per-crew MMK adjustments: Bank Charge, Refund, Deduction
- Grand Total USD + MMK across all crew
- Export buttons:
  - 📄 PDF Summary (full table)
  - 📄 Bank PDF (transfer instructions, MMK)
  - 📄 Cash PDF (signature pickup list)
- 💾 Save persists `rate` and `extraSettings` to `payrollSettings/{billId}`.
